# ADR-004: スクレイピングで初期料理データを構築

## Status
Accepted

## Context
200件以上の家庭料理データをどのように用意するかの判断。
手動入力 / 外部API / スクレイピング / AI一括生成を検討。

## Decision
スクレイピングスクリプト（初回のみ実行）で料理名を収集し、AIでタグを一括付与する運用とする。

## Consequences
- 日本の家庭料理を網羅できる（外部APIは日本料理カバレッジが低い）
- 手動入力コストを削減できる
- スクレイピング対象サイトの利用規約確認が必要
- 初期は個人利用・非公開として構築を進める
- スクリプトは `scripts/scrape/` に置き、本番アプリには含めない
